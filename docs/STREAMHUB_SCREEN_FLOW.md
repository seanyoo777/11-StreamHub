# StreamHub 화면 IA·핵심 사용자 플로우 (기획 2단계)

**문서 버전:** 1.0  
**전제:** 실제 송출 서버 구축, 결제 연결, 토큰 발행, 외부 API 연동, 대규모 코드 작성은 본 단계 범위 밖이다. 화면·플로우·정책 초안만 정리한다.

---

## 1. 전체 IA

### 1.1 최상위 구역

| 구역 | 대상 | 목적 |
|------|------|------|
| **Public** | 비로그인 | 랜딩, 라이브 목록, 방 시청(정책에 따라 제한) |
| **Viewer** | 로그인 시청자 | 팔로우, 채팅, 후원, 신고, 알림 |
| **Creator (BJ)** | BJ 역할 | 대시보드, 방 설정, 송출 안내, 모더레이션, 수익 요약(내부) |
| **Admin** | 운영 | 유저·방·채팅·신고·제재, 배너·추천 슬롯 |

### 1.2 사이트맵(화면 목록)

```
/                          랜딩·추천 라이브
/live                      라이브 목록(카테고리·필터)
/live/[roomId]             방송 시청(플레이어·채팅·후원·신고)
/u/[handle]                BJ/유저 프로필·과거 방(정책 시)

/login, /signup            인증(구현 단계에서 범위 결정)

/creator                   BJ 홈(요약)
/creator/rooms             내 방 목록
/creator/rooms/[id]        방 설정·미리보기 링크
/creator/rooms/[id]/stream 송출 설정(RTMP·키·상태)
/creator/rooms/[id]/mods  모더레이터·금칙어(방 단위)
/creator/insights          시청·후원 요약(내부 데이터)

/me                        내 계정·포인트 잔액·내역
/me/wallet                 포인트/코인 잔액·거래 내역(내부만)
/me/following              팔로우 목록

/admin/*                   관리자(§9)

/legal/*                   약관·운영정책·청소년 보호(정적)
```

### 1.3 내비게이션 원칙

- **시청**은 `/live` 및 `/live/[roomId]`로 진입을 단순화한다.  
- **BJ 작업**은 `/creator` 하위로 몰아 **시청 UX와 시각적 분리**한다.  
- **후원·잔액**은 방 안에서도 가능하되, **원장·내역**은 `/me/wallet`에서 일관되게 본다.

## 2. BJ 방송 시작 플로우

### 2.1 화면 순서(해피 패스)

1. **`/creator`** — 오늘의 방 선택 또는 “새 방 만들기”(정책 시).  
2. **`/creator/rooms/[id]`** — 방 제목, 카테고리, 태그, 썸네일(선택), 연령 표시 저장.  
3. **`/creator/rooms/[id]/stream`** — **RTMP URL**, **스트림 키** 확인·복사·재발급(유출 대비).  
4. BJ PC에서 **OBS/XSplit**에 서버·키 입력 후 **송출 시작**(클라이언트 외부 동작).  
5. 동일 화면 또는 상단 배너에서 **“스트림 수신됨”** 상태 확인(플랫폼이 알 수 있는 경우에 한함; 수신 불가 시 **BJ 수동 “방송 시작”** 버튼으로 대체 가능 — 기획 결정 사항).  
6. **`/live/[roomId]`** — 미리보기 링크로 본인 시청 확인(음소거·채팅 테스트).  
7. **`ON AIR`** 배지 노출, 목록에 라이브 반영.

### 2.2 분기·예외

| 상황 | 화면/동작 |
|------|-----------|
| 키 유출 의심 | `/stream`에서 키 **폐기·재발급**, OBS 재설정 안내 토스트 |
| 송출 끊김 | BJ 대시보드 **경고 배너**, 시청자 쪽 **“일시 중단”** 오버레이 문구 |
| 정책 위반 경고 | `/creator` 또는 모달로 **경고 단계** 표시(운영 정책과 연동) |

### 2.3 금지 범위 준수

- 인제스트 서버를 **직접 구축하지 않는다**. 화면은 **URL·키·상태 표시**와 **가이드 문구**까지만 정의한다.

## 3. 시청자 입장 플로우

### 3.1 해피 패스

1. **`/` 또는 `/live`** — 카드 탭 → **`/live/[roomId]`** 이동.  
2. **플레이어 로드** — 매니페스트 URL·방 메타(제목, BJ, 카테고리).  
3. **채팅 패널** — 로그인 시 전체 참여, 비로그인 시 **읽기 전용 또는 비활성**(정책 선택).  
4. **팔로우** — 로그인 시 버튼 노출, 성공 시 토스트.  
5. **이탈** — 백그라운드 재생 정책은 모바일 OS 제약에 맞춘 안내만(구현 단계).

### 3.2 딥링크·공유

- 공유 URL은 **`/live/[roomId]`** 단일 형태를 권장.  
- 방이 **OFF AIR**이면 **“오프라인”** 플레이스홀더 + BJ 프로필 **`/u/[handle]`** 링크.

### 3.3 분기

| 조건 | UI |
|------|-----|
| 연령·등급 제한 | 진입 전 **확인 모달** 또는 플레이어 가림 |
| 지역 제한(향후) | 안내 페이지로 대체 |

---

## 4. 채팅 플로우

### 4.1 시청자: 메시지 전송

1. **`/live/[roomId]`** 채팅 입력창 포커스.  
2. **전송** — 로그인·채팅 권한 검증 실패 시 **로그인 유도** 또는 **채팅 금지 안내**.  
3. **수신** — 내 메시지는 즉시 낙관적 표시 후 서버 확정 시각/실패 처리.  
4. **후원 하이라이트** — 동일 스트림에 **구분된 메시지 타입**으로 표시(색·아이콘).

### 4.2 BJ: 모더레이션

1. **`/live/[roomId]`** (BJ 본인) 또는 **`/creator/rooms/[id]/mods`** 에서 **유저 카드 메뉴**.  
2. 액션: **타임아웃**, **채팅 금지**, **강퇴(세션 단위)** 등(정책으로 범위 고정).  
3. **결과** — 채팅 스트림에 **시스템 메시지** 또는 조용한 처리(어뷰즈 방지 위해 후자 옵션).

### 4.3 속도 제한·오류

- **레이트 리밋** 초과: 입력창 하단 **쿨다운 문구**.  
- **연결 끊김**: “재연결 중…” 배너, 자동 재구독(구현 시).

---

## 5. 내부 포인트/후원 플로우

### 5.1 전제

- **실제 결제 없음**. 잔액은 **가입 보너스·이벤트·운영 지급** 등 내부 순환으로만 증가한다.  
- **토큰 실발행 없음**. “코인”이 있더라도 **DB 원장** 상의 단위로만 취급.

### 5.2 시청자 → BJ 후원(해피 패스)

1. **`/live/[roomId]`** 에서 **후원** 버튼 또는 선물 아이콘.  
2. **금액 프리셋** + (선택) **짧은 메시지**.  
3. **잔액 확인** — 부족 시 **`/me/wallet`** 으로 이동 안내(“충전 없음 — 이벤트 참여” 등 문구만).  
4. **확인 모달** — 수수료·BJ 수령액 **투명 표시**(내부 정책 수치는 기획 확정).  
5. **전송 성공** — 채팅에 **후원 하이라이트**, BJ 화면에 **알림**(토스트/사이드).  
6. **`/me/wallet`** 에 **거래 내역** 한 줄 추가.

### 5.3 BJ 확인

- **`/creator/insights`** 또는 방 송출 화면 사이드에서 **세션 누적 후원** 실시간 갱신(폴링·WS는 구현 단계).  
- **정산**은 “내부 잔액” 표시만; 현금화 플로우는 문서 밖.

## 6. 신고/제재 플로우

### 6.1 시청자 신고

1. **`/live/[roomId]`** 채팅 행 또는 플레이어 메뉴 **⋯** → **신고**.  
2. **유형 선택**: 스팸, 혐오, 성인/폭력, 저작권, 기타.  
3. **근거**: (선택) 메시지 ID 자동 첨부, 짧은 설명.  
4. **제출 완료** — “접수되었습니다” 토스트, **중복 신고 쿨다운**.

### 6.2 BJ 신고

- 동일 UI에서 **시청자** 대상 신고 또는 **모더레이션으로 즉시 처리** 후 **운영 큐에 전달**(정책 선택).

### 6.3 운영 제재(화면 수준)

- 관리자가 **경고·채팅 정지·방송 중지·계정 정지** 시, 해당 유저/방은 **다음 로그인 또는 다음 액션**에서 **제재 사유 요약** 모달(§9와 연동).

### 6.4 신고자 피드백

- 개인정보 보호를 위해 **개별 처리 결과 비공개**가 기본; **“조치됨”** 수준의 익명 피드백만 선택적.

## 7. 모바일/PC 화면 차이

| 요소 | 모바일 | PC |
|------|--------|-----|
| 레이아웃 | **세로**: 플레이어 상단, 채팅 **탭/바텀시트** | **가로**: 플레이어 좌·채팅 우 **고정 분할** |
| 후원 | 하단 **시트**로 금액·확인 | 플레이어 옆 **패널** 또는 모달 |
| BJ 송출 설정 | **접기 가능한 카드** + 복사 버튼 강조 | **한 화면에 URL·키·가이드** 나란히 |
| 네비 | 하단 탭: 홈 / 탐색 / 팔로우 / 내정보 | 상단 글로벌 네비 + 서브메뉴 |
| 키보드 | 채팅 시 플레이어 **축소 또는 오버레이** | 채팅창 높이 고정, 플레이어 유지 |
| 관리자 | **테이블 스크롤·필터 축소** | **와이드 테이블·키보드 탐색** |

공통: **`/live/[roomId]`** URL은 동일하고 **반응형 브레이크포인트**로 위 레이아웃만 전환한다.

## 8. GameHub/OneAI 연동 후보

> 외부 API 연동은 본 단계에서 하지 않는다. **화면·진입점·카피**만 후보로 남긴다.

### 8.1 GameHub(가칭)

| 후보 | 위치 | 사용자 가치 |
|------|------|-------------|
| **게임 태그** | 방 설정·라이브 카드 | 검색·추천 정확도 |
| **“게임 중” 배지** | `/live/[roomId]` 플레이어 상단 | 콘텐츠 기대치 명확화 |
| **오버레이 URL** (선택 필드) | BJ 송출 설정 | OBS 브라우저 소스로 **게임 HUD** 연계(데이터는 외부 블랙박스) |
| **이벤트 하이라이트** | 채팅 또는 사이드 패널 | “킬/랭크” 등 **저빈도** 텍스트 이벤트(스키마는 별도 RFC) |

### 8.2 OneAI(가칭)

| 후보 | 위치 | 사용자 가치 |
|------|------|-------------|
| **방송 요약(종료 후)** | BJ 인사이트 또는 방 아카이브 자리 | VOD 없을 때 **텍스트 하이라이트** |
| **채팅 요약 버튼** | BJ 전용 패널 | 장시간 방송 **모더레이션 보조** |
| **시청자 Q&A 스택** | (선택) 채팅 옆 | 질문 **끌어올리기** UI만 정의 |

모든 AI 기능은 **옵트인·저빈도·비실시간 우선**을 기획 원칙으로 둔다.

## 9. 관리자 화면 후보

| 화면 | 기능 요약 |
|------|-----------|
| `/admin/dashboard` | 신고 대기 수, 라이브 수, 이상 후원 알림 요약 |
| `/admin/reports` | 신고 큐, 필터, 상세, **조치** 버튼 |
| `/admin/users` | 검색, 역할 변경, 정지, 로그(최소) |
| `/admin/rooms` | 강제 종료, 노출 제한, 카테고리 수정 |
| `/admin/chat` | 전역 금칙어, 특정 방 채팅 잠금 |
| `/admin/banners` | 홈·라이브 상단 공지 |
| `/admin/fees` | 내부 수수료율 **표시·실험 플래그**(실결제 없음) |

접근은 **별도 역할·IP 제한·감사 로그**를 전제로 한다(구현 단계).

## 10. MVP 우선순위

우선순위는 **“시청 루프 완성 → 상호작용 → BJ 도구 → 운영”** 순이다.

| 순위 | 범위 | 포함 화면/플로우 |
|------|------|------------------|
| **P0** | 시청 최소 | `/live`, `/live/[roomId]` 플레이어·방 메타·OFF AIR 처리 |
| **P1** | 채팅 | 로그인 채팅 전송·수신, 기본 레이트 리밋 |
| **P2** | 내부 후원 | 후원 모달·잔액 부족 안내·채팅 하이라이트·`/me/wallet` 내역 |
| **P3** | BJ 송출 안내 | `/creator/rooms/[id]/stream` RTMP·키·가이드·(선택) ON AIR 전환 |
| **P4** | 신고 | 시청자 신고 모달·접수 완료 |
| **P5** | BJ 모더레이션 | 타임아웃·금지 등 최소 1~2개 액션 |
| **P6** | 관리자 | `/admin/reports` + `/admin/users` 최소 조치 |
| **P7** | 연동 자리 | 게임 태그, AI 버튼 **비활성/Coming soon** 라벨만 |

**의도적 후순위:** 푸시 알림, VOD, 유료 구독, 복수 카메라, 저지연 전용 모드.
